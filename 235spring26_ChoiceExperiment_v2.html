Welcome

Scenario: Your county is considering improvements to a popular recreation lake. Currently the lake has murky water, 2 miles of unmaintained trails, and no fish stocking program. The county wants to know which improvements residents value most and how much they'd be willing to pay through an annual household fee.

You will be shown 6 choice occasions. In each, you'll see two possible improvement options (Site A and Site B) and an opt-out (keep the status quo). Choose the option you prefer most.

Each option is described by four attributes:

Discussion Questions

1. Your responses were hypothetical — no real money changed hands. How might hypothetical bias affect these WTP estimates? Would you expect them to be too high or too low?

2. We estimated a conditional logit, which assumes independence of irrelevant alternatives (IIA). What does IIA mean, and when might it be violated in environmental choice experiments?

3. The fee attribute lets us calculate WTP in dollars. What if we hadn't included a monetary attribute — could we still estimate WTP? What would we lose?

4. How does this choice experiment approach compare to the travel cost method from earlier in the course? What types of value can choice experiments capture that revealed preference methods cannot?

5. We estimated a single set of β coefficients for the whole class. But preferences clearly differ. What model extension would allow for preference heterogeneity, and how would it change our WTP estimates?

Setup Instructions (one-time)

Step 1 — Create a Google Sheet: Go to sheets.google.com, create a new blank spreadsheet, and name it "235 Choice Experiment." In row 1, add these headers: Timestamp, StudentID, Choice1, Choice2, Choice3, Choice4, Choice5, Choice6.

Step 2 — Add the Apps Script: In the spreadsheet, go to Extensions → Apps Script. Delete the default code and paste the following:

function doPost(e) {
  var lock = LockService.getScriptLock();
  lock.tryLock(10000);
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var data = JSON.parse(e.postData.contents);
  sheet.appendRow([
    new Date().toISOString(),
    data.studentId,
    data.choices[0], data.choices[1], data.choices[2],
    data.choices[3], data.choices[4], data.choices[5]
  ]);
  lock.releaseLock();
  return ContentService.createTextOutput(
    JSON.stringify({status:'ok'})
  ).setMimeType(ContentService.MimeType.JSON);
}

function doGet(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var data = sheet.getDataRange().getValues();
  var rows = [];
  for (var i = 1; i < data.length; i++) {
    rows.push({
      studentId: data[i][1],
      choices: [
        parseInt(data[i][2]), parseInt(data[i][3]),
        parseInt(data[i][4]), parseInt(data[i][5]),
        parseInt(data[i][6]), parseInt(data[i][7])
      ]
    });
  }
  var json = JSON.stringify(rows);
  var cb = (e.parameter || {}).callback;
  if (cb) {
    return ContentService.createTextOutput(cb + '(' + json + ')')
      .setMimeType(ContentService.MimeType.JAVASCRIPT);
  }
  return ContentService.createTextOutput(json)
    .setMimeType(ContentService.MimeType.JSON);
}

Step 3 — Deploy: Click Deploy → New deployment. Set type to Web app. Set "Execute as" to Me and "Who has access" to Anyone. Click Deploy. Authorize when prompted (click through the "unsafe app" warning — it's your own script). Copy the Web app URL.

Step 4 — Configure this page: Open this HTML file in a text editor. Find the line near the top of the <script> section that says:
const SCRIPT_URL = 'https://script.google.com/macros/s/AKfycbx0CB403nEK0YXCd8_mJbKdaKs3pVxW0pOi3NowgkeauHregB8LiYHkRTAilBg6WbU6/exec';
Replace the placeholder with your URL. Save.

Step 5 — Host & share: Upload to GitHub Pages (free) or any web host. Students open the page URL to take the experiment. You open the same URL with ?mode=instructor appended to see the live dashboard.

To reset between sections: Delete the data rows in your Google Sheet (keep the header). Refresh the instructor dashboard.
